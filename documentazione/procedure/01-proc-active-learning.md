## ğŸ”„ Ciclo di Applicazione (per ogni Knowledge Point)

### Step 1: Introduzione Minima (2-3 minuti)
```
INPUT:  Concetto da insegnare
OUTPUT: Spiegazione essenziale + 1 esempio svolto
REGOLA: Nessuna teoria oltre il minimo necessario
```

**Azione pratica:**
- Presenta il concetto in max 3 frasi
- Mostra UN esempio svolto passo-passo
- Non aggiungere casi particolari (verranno dopo)

### Step 2: Attivazione Immediata (5-10 minuti)
```
INPUT:  Esempio appena mostrato
OUTPUT: 2-5 problemi simili da risolvere
REGOLA: Lo studente deve iniziare SUBITO
```

**Azione pratica:**
- "Ora prova tu" (senza pausa)
- Fornisci problemi dello stesso tipo
- Lo studente lavora INDIVIDUALMENTE

### Step 3: Feedback Istantaneo
```
INPUT:  Risposta dello studente
OUTPUT: Corretto/Errato + Soluzione se sbagliato
REGOLA: Nessun "non so", deve provare
```

**Azione pratica:**
- Verifica immediata (non alla fine della lezione)
- Se sbagliato: mostra soluzione + riprova con problema simile
- Se corretto: passa al Knowledge Point successivo

### Step 4: Progressione
```
INPUT:  Padronanza dimostrata sul KP corrente
OUTPUT: Passaggio al KP successivo (leggermente piÃ¹ difficile)
REGOLA: No avanzamento senza padronanza
```
